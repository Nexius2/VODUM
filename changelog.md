# Changelog

All notable changes to Vodum will be documented in this file.

---

### Fixed
- Fixed Plex websocket initialization on some installations. Added compatibility alias for `DATABASE_PATH` to prevent startup errors and ensure the Plex websocket engine starts correctly.
- Fixed a syntax error in the update checker that could prevent the update status task from running correctly.
- Fixed telemetry update-delay reporting by restoring JSON parsing for `update_status.json`.
- Forced the update checker to run at application startup so the update button is cleared immediately after upgrading Vodum.
- Fixed Plex WebSocket live session refresh logic causing repeated `Unable to refresh live sessions` errors in the logs.
- Updated live session refresh to use the central monitoring collector, ensuring consistent session synchronization across the application.
- Improved server health check logging for Jellyfin servers that are offline or temporarily unreachable.
- Reduced unnecessary ERROR log entries generated by expected Jellyfin connection failures, keeping the logs cleaner and easier to troubleshoot.
- Improved monitoring reliability and stability during server status checks and live session updates.
- Improved Plex stream termination handling.
- Prevented false error logs when Plex reports a session as already terminated (HTTP 404).
- Stream Enforcer now treats already-closed sessions as a successful enforcement action.
- Reduced unnecessary error noise in application logs during subscription enforcement.
- Fixed expired subscription enforcement latency.
- System-managed expired subscription policies now terminate playback immediately.
- Removed the warning/recheck delay for users blocked by subscription expiration.
- Improved Stream Enforcer behavior so expired users cannot keep restarting playback between delayed kills.
- Fixed a timing issue where newly started sessions from expired users could bypass immediate enforcement checks.
- Improved handling of expired subscription policies by evaluating fresh sessions without the standard stabilization delay.
- Fixed an issue where newly started sessions from expired users could bypass immediate enforcement checks.
- Fixed policy enforcement evaluation for fresh sessions by removing the stabilization delay for expired subscription policies.
- Fixed Policy View toggle behavior to correctly switch between grouped and event-based views.
- Fixed Users status filter behavior:
  - "Select All" now correctly reloads and displays all matching users.
  - "Select None" now correctly returns an empty result set.
  - Improved synchronization between filter state and displayed results.

- Fixed Referrals status filter behavior:
  - Fixed inconsistencies between selected statuses and displayed referrals.
  - Restored support for all referral statuses in filtering.
  - Fixed "Select All" and "Select None" actions.
  - Corrected referral counters and filtering logic to prevent missing entries.
  - Fixed Monitoring user search to correctly match linked media usernames and emails.

### Improved
- Reduced log noise in normal mode by moving repetitive maintenance and idle-task messages to debug-only logging.
- Streamlined task logging to focus on task execution, user actions, policy enforcement, warnings and errors.
- Background tasks now generate significantly fewer informational entries when no action is required.
- Improved readability of application logs by hiding routine "no-op" activity outside of debug mode.
- Update availability tooltips now display how long an update has been pending.
- Anonymous telemetry now bypasses the 7-day send delay when debug mode is enabled, making telemetry testing easier.
- Reduced potential server load by keeping the normal 60-second enforcement interval during standard operation.
- Improved expired subscription enforcement responsiveness without permanently increasing task frequency.
- Enhanced protection against repeated playback attempts from expired users while minimizing impact on normal users.
- Improved policy monitoring usability on larger installations by providing a consolidated user-centric view alongside the existing event view.
- Reduced visual noise caused by repeated warn/kill events from the same users.
- Simplified view switching with a single toggle button that dynamically switches between Event View and Group by User.
- Enhanced expired subscription enforcement responsiveness without permanently increasing background task frequency.
- Improved filtering consistency across Users and Referrals tabs.
- Improved search consistency across Subscriptions, Policies, Monitoring and user referral selection.
- Extended user searches to include first name, last name, secondary email, Discord name and linked Plex/Jellyfin identities.
- Improved dashboard quote refresh handling when media servers are offline or temporarily unavailable.
- Prevented unnecessary error logs for unreachable Jellyfin servers by skipping offline instances and downgrading connection failures to warnings.
- Reduced log noise during internet outages, server maintenance, or scheduled shutdowns.

### Added
- Added update aging detection based on the number of days an update has been available.
- Update notifications now become red after 7 days and pulse after 30 days without updating.
- Added update delay metrics to anonymous telemetry to better understand update adoption across active installations.
- Added adaptive Stream Enforcer acceleration for expired subscriptions.
- Stream Enforcer now automatically switches to a 15-second interval when an expired user repeatedly attempts to bypass playback restrictions.
- Added automatic return to the normal 60-second interval after 30 minutes of inactivity.
- Added tracking of repeated enforcement actions for expired subscription policies.
- Added a new **Group by user** view in Monitoring → Policies to quickly identify users generating the most policy violations.
- Added per-user enforcement statistics including warn count, kill count, total actions, last action, last server, and last triggered rule.
- Added a detailed enforcement history modal accessible directly from grouped users.
- Added direct navigation from grouped enforcement history to the existing enforcement details view.
- Added adaptive Stream Enforcer acceleration for expired subscriptions. Repeated playback attempts from expired users can now temporarily increase enforcement frequency.

## Subscription Plans
- Redesigned the Subscription Plans interface to make plan creation and management more intuitive.
- Added a simplified editor focused on the most common subscription limits (streams, IPs, bitrate and devices).
- Added an Advanced mode to access the full policy editor without leaving the plan editor.
- Improved plan summaries and visual presentation to better highlight plan limits and restrictions.
- Added an "Enabled only" filter for subscription plans and persisted its state between sessions.
- Renamed subscription tabs to improve clarity and navigation.
- Added missing multilingual translations across the subscription management pages.
- Improved scrolling and layout behavior for a more consistent user experience.

### Communications
- Renamed **Templates** tab to **Messages** and **Configuration** to **Settings** across the interface and translations.
- Added a new built-in **Stream blocked** system message used when playback is stopped by a policy.
- Stream blocked messages are now automatically sent through the configured communication channels (Email / Discord).
- Added new variables for playback-block notifications:
  - `{policy_name}`
  - `{policy_reason}`
  - `{media_title}`
  - `{server_name}`
  - `{client_name}`
  - `{device_name}`
  - `{blocked_at}`
- Added `{brand_name}` variable to communication templates.
- Default communication templates now use `{brand_name}` instead of hardcoded "VODUM Team".
- Stream blocked system messages are protected:
  - Trigger is locked.
  - Media target is locked.
  - Subscription target is locked.
  - Delay is locked.
  - Delete and New actions are hidden.
- Locked fields are now visually disabled to make system-managed messages easier to identify.
- Campaigns now support advanced audience targeting.
- Added media provider filtering for campaigns:
  - All media servers
  - Plex only
  - Jellyfin only
- Added subscription targeting for campaigns:
  - No subscription filter
  - All subscriptions
  - Specific subscription plan
- Campaign recipient selection now respects server, media provider, and subscription filters.
- Campaign list now displays media provider and subscription targeting information for improved visibility.
- Added database support for campaign provider and subscription filters.
- Existing campaigns are automatically migrated and remain fully compatible.

### Subscription Policies
- Expiration modes using playback blocking now automatically enable the Stream blocked communication message.
- Improved integration between subscription expiration policies and communications.

### Referrals
- Fixed referral status filtering issues.
- Fixed Select All / Select None behavior in referral filters.
- Restored proper referral visibility across Active, Archived and All views.

### Users
- Fixed Select All / Select None behavior in user status filters.
- Improved consistency of multi-status filtering and user list refresh.
